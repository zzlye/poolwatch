# 安卓应用安装与发布

## 安装应用

1. 打开项目的 GitHub“Releases”页面。
2. 下载最新 `android-v*` 版本中的 `.apk` 文件。
3. 在手机中打开安装包，并按系统提示允许本次安装。
4. 启动“号池监控”，登录现有管理员账号。
5. 允许通知权限，开启“持续监控”，再发送一次测试通知。
6. 按应用内提示允许后台运行，并根据手机品牌设置自启动和忽略电池优化。

升级时下载较新的安装包并直接安装，应用会保留已有登录和监控设置。安装包必须使用同一签名；更换签名后需要先卸载旧版。

## 自动检查

当 `main` 分支或拉取请求中的安卓相关文件发生变化时，GitHub Actions 会自动执行单元测试、代码检查和调试版构建。构建完成后，可在对应工作流记录的“Artifacts”区域下载调试安装包。

调试安装包仅用于测试。正式使用请下载 GitHub Release 中的签名安装包。

## 配置发布签名

在 GitHub 仓库的“Settings → Secrets and variables → Actions”中添加以下仓库 Secrets：

| Secret | 内容 |
| --- | --- |
| `ANDROID_KEYSTORE_BASE64` | 签名文件经过 Base64 编码后的完整内容 |
| `ANDROID_KEYSTORE_PASSWORD` | 签名文件密码 |
| `ANDROID_KEY_ALIAS` | 签名别名 |
| `ANDROID_KEY_PASSWORD` | 签名别名密码 |

在 Windows PowerShell 中可使用以下命令生成 Base64 文本：

```powershell
[Convert]::ToBase64String([IO.File]::ReadAllBytes("D:\tools\android-signing\poolwatch-release.jks")) | Set-Content -NoNewline "D:\tmp\poolwatch-keystore-base64.txt"
```

将 `D:\tmp\poolwatch-keystore-base64.txt` 的完整内容保存到 `ANDROID_KEYSTORE_BASE64`。签名文件和密码需要长期离线备份；后续版本必须继续使用同一签名。

## 发布正式版本

1. 确认安卓应用版本号已经更新，测试和检查均通过。
2. 创建并推送格式为 `android-v版本号` 的标签，例如：

   ```powershell
   git tag android-v1.0.0
   git push origin android-v1.0.0
   ```

3. GitHub Actions 会读取签名 Secrets，构建签名安装包并生成 SHA-256 校验文件。
4. 工作流完成后，会自动创建同名 GitHub Release，并附上 `.apk` 与 `.sha256` 文件。
5. 下载两个文件，在本地核对校验值后再分发安装包：

   ```powershell
   Get-FileHash .\poolwatch-android-v1.0.0.apk -Algorithm SHA256
   ```

## 发布失败排查

- 提示缺少 Secret：检查四个 Secret 的名称和内容是否完整。
- 提示签名文件格式错误：重新从原始签名文件生成 Base64 文本，复制时不要加入引号或额外空格。
- 提示密码或别名错误：使用签名文件原始密码和别名重新填写 Secrets。
- 没有触发发布：确认标签以 `android-v` 开头，并已推送到 GitHub。
- 新版无法覆盖安装：确认构建时使用的签名文件与旧版完全相同。
