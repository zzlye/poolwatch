# 安卓应用安装与发布

## 安装应用

1. 打开项目的 GitHub“Releases”页面。
2. 下载最新 `android-v*` 版本中的 `.apk` 文件。
3. 在手机中打开安装包，并按系统提示允许本次安装。
4. 启动“号池监控”，登录现有管理员账号。
5. 允许通知权限，开启“持续监控”，再发送一次测试通知。
6. 按应用内提示允许后台运行，并根据手机品牌设置自启动和忽略电池优化。

`1.1.2` 及更早版本需要手动安装一次 `1.1.3`。从 `1.1.3` 开始，应用会在启动时静默检查更新，也可以在“安卓应用设置 → 应用更新”中手动检查、下载并安装。升级会保留已有登录和监控设置。安装包必须使用同一签名；更换签名后需要先卸载旧版。

## 后台提醒说明

- 开启“持续监控”后，应用会显示常驻通知并接收实时告警。
- 服务端连接长时间没有数据时，应用会自动断开并重新连接；网页退出或登录失效后，也会停止旧连接并提示重新登录。
- Android 12 及更高版本会限制应用从后台重新启动持续监控。系统暂不允许恢复时，应用仍会通过约十五分钟一次的检查补拉告警；再次打开应用后会立即尝试恢复实时监控。
- 手机重启或应用升级后，应用会恢复定时检查，并在系统允许时恢复持续监控。
- 在系统设置中“强行停止”应用后，系统会同时暂停通知、定时检查和开机恢复，需要手动打开一次应用。
- 不同手机品牌还可能限制自启动。建议同时允许通知、自启动、后台运行并忽略电池优化，再使用应用内“测试通知”确认设置。

## 自动检查

当 `main` 分支或拉取请求中的安卓相关文件发生变化时，GitHub Actions 会自动执行单元测试、代码检查和调试版构建。构建完成后，可在对应工作流记录的“Artifacts”区域下载调试安装包。

调试安装包仅用于测试。正式使用请下载 GitHub Release 中的签名安装包。

## 配置发布签名

在 GitHub 仓库的“Settings → Secrets and variables → Actions”中添加以下仓库 Secrets：

| Secret | 内容 |
| --- | --- |
| `ANDROID_KEYSTORE_BASE64` | 签名文件经过 Base64 编码后的完整内容 |
| `ANDROID_KEYSTORE_PASSWORD` | 签名文件密码 |
| `ANDROID_KEY_ALIAS` | 签名别名 |
| `ANDROID_KEY_PASSWORD` | 签名别名密码 |

在 Windows PowerShell 中可使用以下命令生成 Base64 文本：

```powershell
[Convert]::ToBase64String([IO.File]::ReadAllBytes("D:\tools\android-signing\poolwatch-release.jks")) | Set-Content -NoNewline "D:\tmp\poolwatch-keystore-base64.txt"
```

将 `D:\tmp\poolwatch-keystore-base64.txt` 的完整内容保存到 `ANDROID_KEYSTORE_BASE64`。签名文件和密码需要长期离线备份；后续版本必须继续使用同一签名。

正式签名证书的 SHA-256 指纹应为 `B2:AC:C6:D3:08:DB:C3:F6:AC:19:FE:C6:50:E4:EB:34:FB:2D:75:C5:5E:84:FF:71:C5:9E:2A:0E:69:FC:69:68`。发布前应核对安装包证书，避免使用其他项目的签名文件导致旧版本无法直接升级。

## 发布正式版本

1. 确认安卓应用版本号已经更新，测试和检查均通过。
2. 创建并推送格式为 `android-v版本号` 的标签，例如当前版本 `1.1.3`：

   ```powershell
   git tag android-v1.1.3
   git push origin android-v1.1.3
   ```

3. GitHub Actions 会读取签名 Secrets，构建签名安装包并生成 SHA-256 校验文件。
4. 工作流先创建不可见的草稿 Release，补齐 `.apk`、`.sha256` 与固定名称 `android-update.json` 后再正式发布，避免用户看到文件不完整的版本。
5. 应用读取更新清单判断是否有新版本，在用户确认后下载、校验安装包并调用系统安装器；首次安装时系统可能要求允许“安装未知应用”。
6. 下载文件，在本地核对校验值后再分发安装包：

   ```powershell
   Get-FileHash .\poolwatch-android-v1.1.3.apk -Algorithm SHA256
   ```

`android-update.json` 会记录版本号、下载地址、文件大小、SHA-256、更新说明和发布地址。每次发布都使用同一字段和文件名，便于手机端稳定检查更新；安装包仍使用 `poolwatch-android-v版本号.apk` 命名，旧版本下载地址继续有效。

应用在线检查更新时访问当前服务器的 `GET /api/app/update/android`。返回清单中的 `downloadUrl` 使用同源的 `/api/app/update/android/apk?versionCode=版本号` 代理地址，服务器会转发经过校验的 GitHub 安装包，并支持断点续传；手机端无需直接访问 GitHub。

## 发布失败排查

- 提示缺少 Secret：检查四个 Secret 的名称和内容是否完整。
- 提示签名文件格式错误：重新从原始签名文件生成 Base64 文本，复制时不要加入引号或额外空格。
- 提示密码或别名错误：使用签名文件原始密码和别名重新填写 Secrets。
- 没有触发发布：确认标签以 `android-v` 开头，并已推送到 GitHub。
- 新版无法覆盖安装：确认构建时使用的签名文件与旧版完全相同。
